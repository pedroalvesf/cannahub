## Codebase Patterns
- Prisma generate requires `PRISMA_SCHEMA_ENGINE_BINARY=/dev/null` workaround when no network available (engine download blocked)
- Prisma client output configured at `../src/generated/prisma` (gitignored), must run `prisma generate` before build/test
- `package.json` root needs `pnpm.onlyBuiltDependencies` to allow prisma engine postinstall scripts
- Use `npx tsc --noEmit` in `apps/api` or `apps/web` to typecheck
- Domain entities use Entity base class with `static create()` factory, `UniqueEntityID`, and `Optional<>` type for optional props
- Use cases are `@Injectable()` classes with `execute()` returning `Either<Error, Result>` â€” never throw
- Abstract repository â†’ concrete Prisma implementation registered in `DatabaseModule` with `provide/useClass` pattern
- Controllers use `@UseGuards(JwtAuthGuard)`, `@CurrentUser()` for userId extraction via `user.sub`
- DTOs use `class-validator` decorators; registered in `http.module.ts` controllers+providers arrays
- Frontend hooks follow `useQuery`/`useMutation` pattern with `queryClient.invalidateQueries` on mutation success
- Labels live in `apps/web/src/constants/labels.ts` â€” never duplicate
- Shared package exports from `packages/shared/src/index.ts`
- `pnpm build` in `apps/web` runs tsc + vite build (verifies compilation + bundling)
